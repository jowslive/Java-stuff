-Campos,construtores e métodos
	
	O código fonte de uma classe pode ser dividido em duas partes principais : um pequeno envoltório externo que simplesmente nomeia a classe e uma parte interna maior que faz todo o trabalho.

	A parte interna da classe é onde definimos campos (atributos) , construtores e métodos.
	
	Os campos armazenam dados para o uso de cada objeto.
       	
        Os construtores permitem que cada objeto seja configurado adequadamente quando ele é criado.

	Os métodos implementam o comportamento dos objetos.

	Em Java existem pouquissimas regras sobre a ordem que você escolhe para definir campos , construtores e métodos em uma classe.
	
        Nós escolhemos listar os campos primeiro,em segundo os construtores e por fim os métodos.
         	
 		public class NomeDaClasse{
                //campos
                //construtores
                //métodos
                }
        Os campos (atributos) são pequenos nas quantidades de espaço dentro de um objeto que podem ser utilizados para armazenar valores.
        
        Os comentários são inseridos no código fonte de uma classe para fornecer explicações a leitores humanos,eles não tem nenhum efeito sobre o funcionamento da classe.

	Um comentário de uma única linha é introduzido pelos caracteres "//" (sem espaço entre eles).Comentários de múltiplas linhas começam com o par de caracteres "/*" e terminam com o par "*/".

-Construtores

 	Os construtores de uma classe têm um papel especial a cumprir : é sua responsabilidade colocar cada objeto dessa classe , quando ele é criado ,em um estado previamente definido para ser utilizado.

	Um dos recursos característicos dos construtores é que eles têm o mesmo nome da classe em que são definidos 
	
	Ex:  	public class TicketMachine {
                //atributos omitidos
                public TicketMachine(int TicketCost) {
                       price=TicketCost;
                       balance=0;
		       total=0;
                }
		//métodos omitidos
                }

		Os campos de um objeto são inicializados no construtor;outro ponto a observar é que o método construtor nunca retorna nenhum valor.

-Transmitindo dados via parâmetros

	Construtores e métodos recebem valores através de seus parâmetros.

	Ex:	public TicketMachine (int TicketCost)
	
	Distinguimos entre nomes dos parâmetros dentro de um construtor ou método dos valores passados de fora,referindo-nos aos nomes como parâmetros formais e os valores fornecidos pelo usuário são os parâmetros reais.

-Variáveis Locais

	O método refundBalance (abaixo) contém três instruções e uma declaração.A declaração ilustra um novo tipo de variável.
		public int refundBalance(){
                  int amountToRefund;
		  amountToRefund=balance;
		  balance=0
		  return amountToRefund;
                }

	Que tipo de variável é amountToRefund?
	Sabemos que não é um atributo porque os atributos são declarados fora dos métodos,também não é um parametro,pois estes são sempre definidos na assinatura do método.A variável amountToRefund é conhecida como variável local porque é definida dentro de um método.
